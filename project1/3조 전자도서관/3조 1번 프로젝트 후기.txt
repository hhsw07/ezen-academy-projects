3조 1번 프로젝트 전자도서관 후기
1.발표시
	1) 요구사항 정의서를 이야기할땐 각 기능의 이름까지만 설명하고 상세한 기능은 페이지에서 직접 서술하는 것으로 하자
	2) 요구사항 정의서의 내용은 전문적 용어의 사용을 자제하고 간결한 표현을 사용하되 자세하게 서술하자
	3) 발표시 각 담당자가 기능설명하지 않고 한 사람이 통합된 내용을 발표하는 것으로 하자
	4) 요구사항 정의서에서 하지 않은 설명은 페이지에서 하도록 하자

2.PPT 작성시
	1) 웹, 앱의 화면을 한 페이지에 모아서 표현하자
	2) 페이지에서 번호를 통해서 기능을 설명하자, 기능이 한 페이지에 들어가기에 많을 경우는 동일 페이지를 두개로 나눠서 표현하자
	3) 초안에서 만든 내용 중 기획의도의 경우는 삭제하지 말자
	4) 초안에서 만든 내용의 흐름도같은 경우는 남겨두고 수정된 흐름도를 둘다 표현해 진척사항을 표시하자
	5) PPT에서 만든 내용은 합쳤을때 통일성이 있게끔 하자
		- 예시) 회원 정보는 모두 동일한 것을 사용

3. 기능 구현시
	1) 특정 기능의 다양성을 고려해보자
		- 예시) 로그인 화면은 비회원이 마이페이지를 눌러도 갈 수 있고, 메인에서 로그인을 클릭해도 갈 수 있음
	2) 첫 기능 구현의 경우 조금 무리해서 기능을 추가해보되 안되면 중간 단계에서 삭제해보도록 하자
	3) 요구사항 정의서에 담을 기능은 정의서 제작자가 대분류, 중분류정도까지 선정하고 소분류는 각 파트 담당자를 내정해서
		담당자가 기능을 정의(구현)하게 하자
	
3. 향후 프로젝트(2번 프로젝트 관련)
	1) 1주에 화면 설계까지 하고 발표, 2주~3주에 걸쳐서 화면 구현을 하고 발표합니다.(발표는 주마다)
	2) 이번 프로젝트는 실제 화면 구현이 있으므로 구현 단계를 고려해서 주제를 선정합니다.
		
4. 조장 개인의 말
	1) 제가 조원들과 모여서 같이 보내는 시간이 많이 부족한 것 같으니 다음에는 좀 더 소통해볼 수 있도록 하겠습니다.
	2) 사이트에 기능을 넣는다고 넣었는데 위의 소통부재와 더해 너무 개인적으로 진행되서 기능이 충분치 않았던 것 같습니다.
